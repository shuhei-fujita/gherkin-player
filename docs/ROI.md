# ROI: test-run-by-ai 導入効果

> **ステータス: ドラフト（推定値のみ）**
> 以下はすべて推定値。PoC実施後に実測データで書き直す。

## スコープ

```
テスト設計（対象外）→ テスト計画 → テスト実行 → 結果記録 → 進捗管理
```

テスト設計（観点の洗い出し・シナリオ作成）は属人性が高く、対象外。

---

## 工程別の削減率

| 工程       | 従来（手動）                           | 導入後                                     | 削減率  |
| ---------- | -------------------------------------- | ------------------------------------------ | ------- |
| テスト計画 | 手動で実行順序・依存関係を整理         | `/plan-test` で自動生成                    | 70〜80% |
| テスト実行 | ブラウザ手動操作→目視確認→スクショ撮影 | `/playwright-cli` でAI操作・判定・スクショ | 60〜80% |
| 結果記録   | Excelにスクショ貼付、PASS/FAIL手入力   | `task.md` に自動記録                       | 80〜90% |
| 進捗管理   | 手動集計・報告                         | `/plan-test` で自動更新                    | 80〜90% |

---

## 想定ベースの試算（katanaプロジェクト：36シナリオ）

| 工程                     | 手動（推定）              | AI自動（推定）               |
| ------------------------ | ------------------------- | ---------------------------- |
| テスト計画               | 2〜3時間                  | 5〜10分                      |
| テスト実行（36件）       | 12〜18時間（1件20〜30分） | 3〜5時間（1件5〜8分 + HITL） |
| 結果記録・エビデンス整理 | 4〜6時間                  | ほぼ0                        |
| **合計**                 | **18〜27時間**            | **3〜5時間**                 |

**工数削減率: 75〜83%**

## 月次コスト比較

前提: QA時給 5,000円 / 月 5〜10画面 / 1画面 20〜35シナリオ

|            | 手動           | AI自動      | 差分                   |
| ---------- | -------------- | ----------- | ---------------------- |
| 月間工数   | 90〜270h       | 15〜50h     | **-75〜220h**          |
| 月間コスト | 45〜135万円    | 7.5〜25万円 | **-37〜110万円/月**    |
| 年間コスト | 540〜1,620万円 | 90〜300万円 | **-450〜1,320万円/年** |
